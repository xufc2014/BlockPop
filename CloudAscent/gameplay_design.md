# 云端冲天 (CloudAscent) - 游戏玩法设计文档

## 一、游戏概述

**游戏名称：** 鬼谷无双·云端冲天
**游戏类型：** 垂直跳跃 / 无尽攀爬
**核心操作：** 左右方向键控制水平移动，角色自动在平台上弹跳
**核心目标：** 尽可能高地向上攀爬，获取最高分数

---

## 二、基础玩法（已实现）

### 2.1 角色控制
- 角色 **Aquafluff（水云兽）** 自动在踩到平台时弹跳
- **左/右方向键** 控制水平移动（空中也可操作）
- 角色从屏幕左侧飞出会从右侧出现（环绕穿越）

### 2.2 得分机制
- 基础得分 = 攀爬高度 / 8（向下滚动距离累计）
- 实体奖励加分：收集星云宠物 +100 分
- 实体惩罚减分：碰到怪物 -50 分，碰到雷云 -500 分

### 2.3 游戏结束条件
- 角色坠落至屏幕底部以下 → 游戏结束

## 三、平台系统

### 3.1 现有平台类型

| 类型 | 名称 | 外观 | 机制 |
|------|------|------|------|
| 普通板 | 米白板 | 米白色木纹 | 固定不动，正常弹跳 |
| 紫楹板 | 翻倍板 | 紫色发光 | 弹跳高度翻倍 |
| 青蓝板 | 移动板 | 青蓝色 | 左右匀速移动，边界反弹 |
| 玄棕板 | 易碎板 | 深棕色 | 踩上 1 秒后爆碎消失 |

### 3.2 新增平台类型（待实现）

| 类型 | 名称 | 外观 | 机制 |
|------|------|------|------|
| 弹簧板 | 橙色弹簧板 | 橙色+弹簧图案 | 弹跳高度 ×2.5，伴随弹簧动画和音效 |
| 传送板 | 星光紫板 | 星光闪烁紫色 | 踩上后随机传送到屏幕内另一块板上方 |
| 冰板 | 冰蓝透明板 | 半透明冰蓝 | 角色落地后水平滑行，不可停止移动 0.8 秒 |
| 毒雾板 | 绿色毒雾板 | 绿色冒泡 | 踩上后屏幕出现毒雾遮挡效果 2 秒，不影响弹跳 |

## 四、道具系统（新增）

### 4.1 道具概述
在攀爬过程中，随机从平台上方生成可收集的道具，触碰即生效。道具持续一定时间后自动消失。

### 4.2 道具列表

| 道具 | 外观 | 效果 | 持续时间 |
|------|------|------|----------|
| 护盾 | 金色光环 | 免疫一次怪物碰撞伤害 | 单次触发 |
| 磁铁 | 红色磁铁 | 自动吸附附近的星云宠物（扩大拾取范围 3 倍） | 8 秒 |
| 喷射背包 | 蓝色火箭 | 连续 3 次弹跳后获得一次主动喷射（按空格键触发向上冲刺） | 3 次跳跃内有效 |
| 火箭鞋 | 红色火焰鞋 | 弹跳高度 +50%，速度 +30% | 10 秒 |
| 时钟 | 金色时钟 | 减慢所有移动板和追踪怪物的速度 50% | 6 秒 |
| 缩小药水 | 紫色瓶子 | 角色缩小 50%，碰撞体积减小（更容易躲避怪物） | 8 秒 |

### 4.3 道具生成规则
- 道具从平台上方缓缓飘下（下落速度 0.5 px/帧）
- 每隔 15~25 个平台间距生成一个道具
- 同屏最多存在 2 个道具
- 道具碰撞箱略大于视觉大小，便于拾取

## 五、天气系统（新增）

### 5.1 系统概述
随着攀爬高度增加，天气会动态变化，影响视觉效果和部分游戏机制。

### 5.2 天气阶段

| 攀爬高度 | 天气 | 视觉效果 | 游戏影响 |
|----------|------|----------|----------|
| 0 ~ 2000 | 晴天 | 蓝天白云，明亮 | 无 |
| 2000 ~ 5000 | 多云 | 云层增厚，光线变柔和 | 无 |
| 5000 ~ 8000 | 暴风 | 屏幕摇晃，雨滴粒子，闪电特效 | 角色受随机横向风力影响 |
| 8000 ~ 12000 | 雷暴 | 频繁闪电，屏幕闪白，雷声 | 偶尔落雷警告（闪烁区域 1.5 秒后落雷，需躲避） |
| 12000 ~ 18000 | 星空 | 夜空星空背景，流星划过 | 无（视觉奖励阶段） |
| 18000+ | 极光 | 极光色彩飘动，星星闪烁 | 分数倍率 ×1.5 |

### 5.3 风力机制
- 暴风阶段：每帧给角色施加 -0.3 ~ +0.3 的随机水平力
- 风力方向每 3~5 秒变化一次
- 风力通过风向指示器（屏幕顶部小旗帜）提示玩家

## 六、连击系统（新增）

### 6.1 连击机制
- 连续踩到平台不落地（中间没有掉落到屏幕下方）即维持连击
- 连击计数显示在屏幕左上角分数旁边

| 连击数 | 奖励 |
|--------|------|
| 10 连击 | 额外 +50 分 |
| 25 连击 | 额外 +150 分 |
| 50 连击 | 额外 +500 分 |
| 100 连击 | 额外 +2000 分 |
| 每增加 50 连击 | 额外 +2000 分（递增） |

### 6.2 连击特效
- 10 连击：数字变为金色
- 25 连击：数字加外发光效果
- 50 连击：全屏边缘出现火焰光效
- 100 连击：全屏彩虹光效 + 大字提示 "LEGENDARY!"

## 八、难度递增（新增）

### 8.1 难度曲线
随着攀爬高度增加，游戏难度逐步提升：

| 攀爬高度 | 难度变化 |
|----------|----------|
| 0 ~ 3000 | 新手期：只有普通板和少量紫楹板 |
| 3000 ~ 8000 | 过渡期：开始出现移动板和易碎板 |
| 8000 ~ 15000 | 挑战期：移动板增多，易碎板碎裂时间缩短至 0.7 秒 |
| 15000 ~ 25000 | 困难期：追踪怪物增多，平台间距增大，出现冰板 |
| 25000+ | 地狱期：所有平台类型出现，落雷频繁，毒雾板出现 |

### 8.2 怪物难度
- 低高度（0~5000）：怪物移动速度 ×0.8
- 中高度（5000~15000）：怪物正常速度
- 高高度（15000+）：怪物移动速度 ×1.3，追踪怪物出现频率翻倍

## 十一、UI/UX 设计

### 11.1 按钮样式升级
- **渐变背景**：从上到下颜色渐变，替代纯色填充
- **悬停效果**：鼠标悬停时亮度提升 + 轻微放大（1.05 倍）
- **点击反馈**：按下时缩小至 0.95 倍
- **圆角阴影**：按钮下方有 3px 偏移半透明阴影
- **边框高光**：顶部 1px 亮边模拟立体感

### 11.2 暂停功能
- 按 **ESC 键** 或 **P 键** 暂停游戏
- 暂停时弹出半透明遮罩 + 暂停菜单
- 暂停菜单选项：继续游戏 / 设置 / 重新开始 / 退出游戏

### 11.3 开始页面
- 游戏标题 "云端冲天" 大字动画（呼吸发光效果）
- 副标题 "鬼谷无双"
- "开始游戏" 按钮
- "设置" 按钮
- 最高分显示
- 操作说明（可展开/收起）

### 11.4 游戏内 HUD
- 左上角：分数（带文字阴影）
- 分数下方：连击计数（金色高亮）
- 右上角：静音按钮 + 设置齿轮按钮
- 顶部中央：当前高度指示
- 活跃道具状态条（屏幕底部横向排列显示当前生效的道具及剩余时间）

### 11.5 游戏结束页面
- 半透明黑色遮罩
- "游戏结束！" 大字
- 本次得分
- 最高分记录（如有新纪录显示"新纪录！"闪烁）
- 游戏时长
- 解锁的成就（如有）
- "再玩一次" 和 "提交成绩" 按钮

## 十三、文件结构规划

```
CloudAscent/
├── CloudAscent.py          # 主入口（精简后只负责初始化和启动）
├── engine/
│   ├── game.py             # Game 主类
│   ├── cloud.py            # 主角类
│   ├── plank.py            # 平台类（所有类型）
│   ├── entity.py           # 浮空实体类
│   ├── particle.py         # 粒子系统
│   ├── weather.py          # 天气系统
│   ├── items.py            # 道具系统
│   ├── combo.py            # 连击系统
│   ├── audio.py            # 音频管理器
│   └── ui.py               # UI 组件（按钮、对话框、设置页面）
├── scenes/
│   ├── start_screen.py     # 开始页面
│   ├── game_screen.py      # 游戏主场景
│   ├── pause_screen.py     # 暂停页面
│   ├── settings_screen.py  # 设置页面
│   └── gameover_screen.py  # 结束页面
├── data/
│   ├── settings.json       # 用户设置
│   ├── highscore.json      # 最高分记录
│   └── achievements.json   # 成就数据
├── img/                    # 图片资源
├── Sound/                  # 音频资源
└── gameplay_design.md      # 本文档
```

> **注意：** 当前所有代码都在单一文件 `CloudAscent.py` 中。后续重构时可根据需要拆分为模块化结构，但也可以保持单文件实现以降低复杂度（取决于代码量增长情况）。
